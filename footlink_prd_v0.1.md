1. PRD v0.1
1.1 제품명

풋링크 MVP

1.2 제품 한 줄

서울/경기 지역 풋살 유저가 1명 / 2~4명 / 팀 기준으로 앱 진입 직후 자신의 상황에 맞는 경기를 바로 보고, 연락하거나 참가 요청을 보내서 빠르게 매칭할 수 있는 모바일 웹 MVP.

1.3 문제 정의

기존 풋살 매칭 서비스는 대체로 탐색형이다.
유저는 지역, 시간, 유형, 조건을 여러 번 고르며 검색해야 한다.

하지만 실제 니즈는 훨씬 단순하다.

나 혼자인지
몇 명이 있는지
팀인지
지금 근처에서 가능한 게 있는지

즉, 유저는 **“찾고 싶은 것”보다 “지금 가능한 것”**을 알고 싶다.

1.4 목표

이 MVP의 목표는 기능 완성도가 아니라 수요 검증이다.

핵심 질문은 3개다.

유저가 앱 진입 직후 필터 없이도 바로 행동하는가
1명 / 2~4명 / 팀 진입 방식이 실제로 탐색보다 빠른가
인앱 채팅/결제 없이도 참가 요청 또는 오픈채팅 연결만으로 매칭이 성립하는가
1.5 대상 사용자
A. 1명 참가 유저
혼자 풋살하고 싶음
용병이나 개인참가 가능한 경기를 찾음
빠르게 결정하고 싶음
B. 2~4명 소규모 유저
친구 2~4명이 같이 뜀
빈 자리 있는 경기나 부분 합류 가능한 팀을 찾음
“우리 몇 명 들어갈 수 있나”가 중요함
C. 팀 대표
팀 단위로 상대 팀을 찾음
지역, 시간, 상대 레벨이 중요함
팀매치 성사 속도가 중요함
1.6 핵심 가설
가설 1: 첫 화면에서 1명 / 2~4명 / 팀만 고르게 하면 첫 행동까지 걸리는 시간이 짧아진다.
가설 2: 카드에 “시간”보다 “상태”를 먼저 보여주면 클릭률이 올라간다.
가설 3: 지도보다 리스트가 빠르다.
가설 4: 인앱 채팅 없이도 오픈채팅 링크 또는 참가 요청으로 충분히 수요를 검증할 수 있다.
1.7 성공 지표

정식 서비스 지표가 아니라 MVP 검증 지표로 본다.

첫 의미 있는 행동까지의 시간: 20초 이내
피드 카드 클릭률: 30% 이상
상세 → 연락/참가 요청 전환율: 15% 이상
경기 생성 완료율: 60% 이상
생성된 모집글의 24시간 내 첫 응답률: 40% 이상
7일 내 재방문율: 20% 이상

1.11 핵심 사용자 플로우
플로우 A: 1명

앱 진입 → 1명 선택 → 위치 허용 → 개인참가/용병 가능한 경기 피드 노출 → 카드 클릭 → 상세 → 참가 요청 또는 오픈채팅 연결

플로우 B: 2~4명

앱 진입 → 2~4명 선택 → 인원 수 선택(2/3/4) → 위치 허용 → 해당 인원 수 이상 수용 가능한 경기만 노출 → 상세 → 연락/요청

플로우 C: 팀

앱 진입 → 팀 선택 → 위치 허용 → 팀매치만 노출 → 상세 → 연락/매칭 조율

플로우 D: 경기 생성

앱 진입/홈 → 지금 한겜 만들기 → 유형 선택 → 시간/지역/인원/비용/연락 방식 입력 → 게시 → 내 활동에서 상태 확인
